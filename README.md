# frameworks-3

Node.js сервис для задания по настройкам запуска и базовой защите веб-службы.

## Что реализовано

1. Чтение настроек из трех источников: файл, переменные окружения и CLI.
2. Явный приоритет источников: file < env < cli.
3. Ранняя валидация конфигурации до запуска сервера.
4. CORS с доступом только от доверенных источников.
5. Ограничение частоты запросов с разными лимитами по маршрутам.
6. Защитные заголовки: X-Frame-Options, Cache-Control, X-Content-Type-Options.
7. Два режима работы: learning и production.

## Установка

```bash
npm install
```

## Запуск

Учебный режим:

```bash
npm run start:learning
```

Боевой режим:

```bash
npm run start:production
```

Режим переключается только настройкой (в config/default.json или через APP_MODE).

Остальные параметры можно переопределять через env и CLI:

```bash
APP_MODE=production PORT=8080 TRUSTED_ORIGINS=http://localhost:3000 npm start -- --port=9090
```

В примере выше итоговый порт будет 9090, потому что CLI имеет наивысший приоритет для operational-параметров.

## Проверки

```bash
npm test
```

Тесты подтверждают:

1. Приоритет источников настроек.
2. Раннее обнаружение некорректной конфигурации.
3. Блокировку недоверенных Origin.
4. Срабатывание rate limit.
5. Разное поведение learning и production.

## Критичные настройки и почему

1. trustedOrigins: ошибка в этом списке может открыть доступ нежелательным источникам.
2. rateLimit: слишком высокий лимит повышает риск перегрузки, слишком низкий ломает легитимный трафик.
3. mode: влияет на подробность ошибок и профиль ограничений.

Ранняя валидация останавливает запуск до старта сети, поэтому неверная конфигурация не доходит до боевой эксплуатации и не создает скрытые риски.
